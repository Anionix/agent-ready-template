# Maintainers

Current maintainer:

- `Anionix`

## Maintainer responsibilities

- Keep `main` protected.
- Keep CI, the CodeQL advanced workflow, Scorecard, secret scanning, and release-readiness checks healthy.
- Review security reports through GitHub private security advisories.
- Keep release notes and public process documentation current.
- Avoid claiming OpenSSF Best Practices criteria that depend on independent maintainers or reviewers until those roles exist.

## Access continuity

The repository is currently maintained in solo mode. A private emergency
recovery procedure exists outside this public repository so issue handling,
pull-request handling, and source release publication can continue if the
current maintainer is unavailable. The recovery material is deliberately not
stored in this repository.

Adding a second trusted maintainer remains the preferred path before claiming
bus factor, independent-review, or full OpenSSF Best Practices Gold criteria.
