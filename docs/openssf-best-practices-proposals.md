# OpenSSF Best Practices Proposal URLs

These URLs prepare human-reviewed automation proposals for OpenSSF Best
Practices project `12855`.

Project 12855 migrated from old baseline: the old Silver baseline was
`Anionix/agent-ready-repo-template-spec@v0.1.1`, but the active project record
now belongs to `Anionix/agent-ready-template`.

Use these links in the OpenSSF UI. They do not write through the REST API, and
they do not commit `.bestpractices.json`. Each URL includes `overrides=` so
existing old-repo evidence is highlighted for deliberate replacement.

For reusable row-level tracking across repository migrations or badge refreshes,
use `docs/openssf-checksheet-template.md`. The generated project tracking sheet
for this repository is `docs/openssf-checksheet-12855.yml`; the generic CLI
workbench lives at https://github.com/Anionix/ossf-badge-workbench.

## Save Order

1. Open and save the Passing proposal.
2. Open and save the Silver proposals in order.
3. Open and save the Gold review caveats proposal if the UI still has
   unknown or stale review fields.
4. Open and save the Baseline 3 proposal.
5. Read back `https://www.bestpractices.dev/projects/12855.json`.

The Silver security/release proposal sets `signed_releases` to `Met` because
the `v0.1.2` signed release artifacts have been published and verified for
`Anionix/agent-ready-template`.

Do not add a live README badge until the readback confirms migrated Silver and
no active criterion justification depends on the old private repository.

The proposal set intentionally does not claim separate human review. This
repository is operated in solo-maintainer hybrid mode: pull requests and
required checks are used, but routine changes do not require separate human
approval. Keep two-person review unmet unless the operating model changes.
