# Daytona Dedicated-Computer Sandbox Note

Daytona is a future dedicated-computer cloud sandbox adapter reference. It is not required by this template and no Daytona credentials, SDKs, or API calls are added.

## Fit

Evaluate Daytona when a project wants provider-managed development environments or persistent remote sandboxes for agent workflows.

## Adoption checks

- Treat the sandbox as a remote machine with lifecycle, storage, account, and access-policy decisions.
- Store API keys outside the repository.
- Verify persistence, environment variables, logs, network behavior, and deletion semantics.
- Keep this separate from local-first sandbox profiles and default CI.

## Reference

- https://www.daytona.io/docs/
