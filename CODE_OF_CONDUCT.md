# Code of Conduct

This project expects professional, respectful collaboration.

## Expected behavior

- Be respectful and constructive.
- Focus critique on code, documentation, risks, and evidence.
- Avoid harassment, personal attacks, discriminatory language, and private information disclosure.
- Keep security-sensitive reports out of public issues.

## Reporting

- Use GitHub issues or pull requests for ordinary project conduct problems that can be discussed publicly.
- Use GitHub private security advisories for vulnerability reports or reports that include sensitive security details.
- Do not post secrets, credentials, private contact information, or unrelated personal data in public reports.

## Enforcement

The maintainer may edit, hide, lock, or remove inappropriate content, and may block accounts that repeatedly disrupt project work. Enforcement decisions should be documented in the relevant GitHub moderation or security channel when possible.
